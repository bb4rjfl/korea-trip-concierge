# 02. 제품 정의 + 통합 툴 스펙 (Korea Trip Concierge)

> 무엇을 만드는가. 통합 결정 배경과 툴 목록. 입출력 상세 계약은 `03_tool_contracts.md`.

---

## 1. 한 줄 정의
방한 외국인이 한국에서 막히는 지점(장소탐색·외국인 친화 매장·교통·결제)을 **영어로 구조화**해 해결하고, **버튼/선택지를 눌러 이어가는** 올인원 MCP 비서.

## 2. 타깃 & 가치
- **1차 타깃**: 방한 외국인 (월 200만+, 만족도 97.8%인데 '디지털 장벽'이 재방문 저해)
- **2차 타깃**: 외국인 지인을 안내하는 한국인 → **본선 투표 모수 확장**
- **검증된 불편(최근 6개월)**: 교통정보 부족(16.1%) · 가입/인증(13.1%) · 결제(11.5%) · 앱 오류(10.4%) · 지도/장소탐색 장벽. 외국카드 대중교통 결제 비중 1~5%.

## 3. 통합 결정 (왜 한 서버로)
세 아이디어(Concierge / K-Pass Finder / K-Bus Companion)는 모두 **"외국인이 한국에서 막히는 지점"** 이라는 단일 정체성. 분리 시 각 서버가 빈약해지고 KC 2대 제한·툴 3~10개 권장에 불리. **하나의 충실한 서버(8개 툴)** 가 심사(명확성)·본선(데모/투표)에 유리.
→ 결정 이력: `06_decision_log.md` D-001, D-002.

## 4. 🛑 푸시 알림 제약 (반드시 인지)
MCP는 **요청-응답 전용**. 서버가 사용자에게 먼저 알림을 보낼 수 없다. 따라서 K-Bus의 "내리기 전 알림"은 **푸시가 아니라 조회**로 구현한다:
- 사용자: "143번 지금 어디야? 명동성당 몇 정거장 남았어?"
- 서버: "**3정거장 전, 약 6분.** 명동성당 다음다음이에요" + **[🔄 새로고침] [🚏 도착 임박 다시 확인]** 선택지 제공
- 사용자는 버튼 누르듯 짧게 재호출 → "준비 알림" 체감을 조회로 대체.
(본선 Kakao Tools에서 푸시/Widget 가능 여부 별도 검토.)

## 5. 통합 툴 목록 (8개 — 3~10 권장 충족)

| # | 툴명 (영문, kakao 금지) | 역할 | 출처 |
|---|---|---|---|
| 1 | `searchPlaceForeigner` | 자연어 의도 → 장소 추천 (외국인 친화 가중) | Concierge |
| 2 | `findForeignerFriendlyStore` | [인증불필요/해외카드가능/다국어메뉴] 조건 매장 필터 | **K-Pass Finder** |
| 3 | `getTransitRoute` | 출발→도착 대중교통 경로·요금·환승 (영문) | Concierge |
| 4 | `trackBusArrival` | 특정 버스 실시간 위치·남은 정거장 **조회** | **K-Bus Companion** |
| 5 | `explainPayment` | 이 상황에서 외국인 결제 가능 방법 안내 | Concierge |
| 6 | `getAreaGuide` | 동네별 한 줄 가이드 + 핵심 스팟 | Concierge |
| 7 | `translateMenuContext` | 메뉴/음식 맥락 설명 (단순 번역 아님) | Concierge |

> 모든 툴은 **다음 액션 선택지(버튼)** 를 응답 말미에 포함한다 (`04_ux_interaction.md` 표준).

## 6. K-Pass Finder 상세 (아이디어 1 반영)
- 입력 예: "성수동인데 예약 없이 가거나 해외카드로 바로 되는 맛집"
- 필터 플래그: `noReservationNeeded` / `acceptsForeignCard` / `hasMultilingualMenu` / `walkInOk`
- 데이터: 공공 관광/음식점 데이터 + (가능 시) 외국인 결제가능 매장 정보. **캐치테이블 등 예약앱 인증연동은 금지(제3자 인증 → 반려 위험)**. 카카오맵 직접연동은 "되면 좋은 것"으로 분류, 필수 아님.

## 7. K-Bus Companion 상세 (아이디어 2 반영, 조회형)
- 입력 예: "143번 탔고 명동성당에서 내릴래"
- 데이터: 서울시/국토부 **실시간 버스 도착정보 Open API**
- 출력: 남은 정거장 수·예상 시간·하차 정류장 직전 안내 문구(영문) + 새로고침 선택지
- ⚠️ 실시간성 = 고유가치(반려 회피 핵심). 단 API 응답 지연 시 타임아웃·캐싱으로 p99 3s 사수.
