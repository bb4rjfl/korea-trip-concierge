# 08. 외부 API 키 발급 가이드 (비개발자용 클릭 따라하기)

> **목적**: Korea Trip Concierge 서버가 실데이터로 동작하려면 외부 API 키 3종이 필요하다.
> 이 문서는 **개발 지식이 전혀 없어도** 화면에서 무엇을 누르면 되는지 단계별로 안내한다.
> 각 단계의 메뉴명·버튼명은 2026년 6월 기준 실제 화면을 확인해 작성했다(각 절 하단 출처 참조).
> 사이트는 수시로 개편되므로 버튼 위치가 약간 다를 수 있다 — 그럴 땐 **굵게 표시한 키워드**로 찾으면 된다.

## 0. 한눈에 보기 — 왜 이 키들이 필요한가

| .env 변수 | 발급처 | 무엇에 쓰나 (관련 툴) | 발급 난이도 |
|---|---|---|---|
| `BUS_API_KEY` | 공공데이터포털 (data.go.kr) — 국토교통부 TAGO 버스도착정보 | 실시간 버스 도착 정보 안내 (K-Bus Companion 계열 툴) | ★☆☆ 자동승인·즉시 |
| `TOUR_API_KEY` | 공공데이터포털 (data.go.kr) — 한국관광공사 영문 관광정보(EngService2) | 외국인용 영어 관광지/행사/맛집 정보 (장소탐색 툴) | ★☆☆ 자동승인·즉시 |
| `TRANSIT_API_KEY` | ODsay LAB (lab.odsay.com) | 대중교통 환승·길찾기 경로 (경로안내 툴) | ★★☆ 가입 후 앱 등록 |

> 이 서버의 8개 툴 중 **5개**가 위 키에 의존한다. 지식 기반 툴 3종(설명/안내성)은 키 없이도 동작하지만,
> 실시간/연동 가치가 핵심인 위 5종은 키가 있어야 "미연동" 응답을 벗어나 정상 작동한다.

> ⚠️ **공통 주의**: `BUS_API_KEY`와 `TOUR_API_KEY`는 **둘 다 data.go.kr에서 발급**하지만,
> **API마다 "활용신청"을 따로 해야** 한다. 한 번 가입하면 계정은 공용이지만 신청은 데이터별로 별도다.
> (자세한 내용은 §A-7 참조)

## A. 공공데이터포털(data.go.kr) — `BUS_API_KEY` · `TOUR_API_KEY` 공통 절차

`BUS_API_KEY`와 `TOUR_API_KEY`는 모두 **공공데이터포털**이라는 한 사이트에서 받는다.
먼저 회원가입(§A-1~A-2)을 한 번만 해두고, 이후 데이터별 활용신청(§B 버스, §C 관광)을 진행한다.

### A-1. 회원가입

1. 웹브라우저(크롬 권장) 주소창에 **`https://www.data.go.kr`** 입력 후 접속한다.
2. 화면 **오른쪽 위**의 **[회원가입]** 버튼을 클릭한다.
3. 회원 유형 선택 화면이 나오면 **[개인 회원]**(일반 국민) 영역의 **[개인 회원으로 가입하기]**(또는 [가입하기]) 버튼을 누른다.
   - (사업자/법인이 아니라면 개인 회원이면 충분하다.)
4. **이용약관·개인정보 수집 동의** 항목이 나오면 **[전체 동의]** 체크 후 **[다음]**.
5. **본인인증** 화면에서 인증 수단을 고른다. 보통 다음 중 하나다:
   - **휴대폰 본인인증**(가장 빠름, 1~2분) — 통신사 선택 → 이름/주민번호 앞자리/휴대폰번호 입력 → 문자 인증번호 입력
   - **아이핀(I-PIN)** 인증
   - 외국인은 휴대폰 인증이 제한될 수 있어, 가능하면 한국 휴대폰 또는 아이핀을 사용한다.
6. 본인인증이 끝나면 **아이디 / 비밀번호 / 이메일**을 입력하고 **[가입 완료]**.
7. 입력한 이메일로 온 **인증 메일**의 링크를 클릭해 이메일 인증까지 마친다.
   - 전체 소요시간: 보통 **3~5분**.

### A-2. 로그인

1. 다시 `https://www.data.go.kr` 상단 오른쪽 **[로그인]** 클릭.
2. 가입한 아이디/비밀번호 입력 후 로그인. (소셜/간편 로그인도 있으면 사용 가능.)

### A-3. 원하는 API 검색하기

1. 메인 화면 **중앙 상단의 검색창**(돋보기 아이콘)에 키워드를 입력한다.
   - 버스용: **`TAGO 버스도착정보`** 또는 **`버스도착정보`**
   - 관광용: **`한국관광공사 영문 관광정보`** 또는 **`영문 관광정보`**
2. 검색 후 결과 상단의 **[오픈 API]** 탭을 클릭한다(데이터/파일/표준 등 탭 중 하나).
   - 우리가 필요한 것은 파일이 아니라 **실시간 호출용 오픈 API**다. 반드시 [오픈 API] 탭에서 고른다.
3. 검색 결과 목록에서 정확한 데이터명을 클릭해 **상세 페이지**로 들어간다.
   - (구체적 데이터명·세부 절차는 §B, §C에서 안내.)

### A-4. 활용신청하기

1. API 상세 페이지에서 오른쪽(또는 상단)의 파란색 **[활용신청]** 버튼을 클릭한다.
2. **활용신청 양식**이 나온다. 다음을 입력/선택한다:
   - **시스템 유형**: 보통 기본값 **`일반`** 그대로 둔다.
   - **활용목적 선택**: 라디오 버튼 중 **`웹 사이트 개발`** 또는 **`앱개발 (모바일, 솔루션 등)`** 선택.
   - **활용목적 상세**(텍스트칸): 예) `방한 외국인 대상 여행 안내 MCP 서비스 개발 및 테스트`
   - **상세기능정보**: 제공되는 오퍼레이션(예: 정류소별 도착예정정보 조회)에 체크가 되어 있는지 확인. 기본 전체 선택돼 있으면 그대로 둔다.
   - **라이선스 표시 동의**: **[동의합니다]** 체크.
3. 맨 아래 **[활용신청]**(또는 [신청]) 버튼을 누른다.

### A-5. 자동승인 vs 심사승인 — 승인까지 걸리는 시간

- 공공데이터포털 API는 두 단계 표시가 있다: **개발단계 / 운영단계**.
- **자동승인** API는 활용신청 즉시 승인되어 바로 키를 쓸 수 있다.
  - 다행히 **TAGO 버스도착정보·한국관광공사 영문 관광정보 모두 "개발단계: 자동승인 / 운영단계: 자동승인"** 이라 즉시 사용 가능하다.
- **심사승인** API라면 제공기관 담당자가 검토 후 승인하며 보통 **1~2시간 ~ 수일**이 걸린다.
- ⚠️ 자동승인이어도 **발급된 서비스키가 실제 호출에서 인식되기까지 보통 1~2시간(최대 24시간)** 지연이 있을 수 있다. 키를 넣었는데 `SERVICE_KEY_IS_NOT_REGISTERED_ERROR` 가 뜨면 조금 기다렸다 다시 시도한다.

### A-6. 발급된 인증키(서비스키) 확인 위치

1. 로그인 상태에서 상단 오른쪽 **[마이페이지]** 클릭.
2. 왼쪽 메뉴에서 **[오픈 API] → [개발계정]** 클릭.
3. 신청한 API 목록이 보인다. 해당 API명을 클릭해 들어가면 **[일반 인증키(서비스키)]** 항목이 있다.
   - 여기 표시되는 긴 문자열이 바로 `.env`에 넣을 키다.
   - 이 화면에서 **호출 주소(End Point)**, **요청 변수**, **일일 트래픽 한도**도 함께 확인할 수 있다.

### A-7. ⚠️ Encoding 키 vs Decoding 키 — 어느 걸 써야 하나

마이페이지 개발계정 화면에는 인증키가 보통 **두 가지**로 보인다:
- **Encoding(인코딩) 인증키** — URL에 바로 붙이도록 미리 인코딩된 형태(`%2B`, `%3D` 등이 섞임)
- **Decoding(디코딩) 인증키** — 원본 그대로의 형태(`+`, `=` 등 그대로)

**이 프로젝트에서는 Decoding(디코딩) 인증키를 쓰는 것을 권장한다.**
- 이유: 우리 서버 코드는 요청을 보낼 때 `URLSearchParams`로 파라미터를 **자동 인코딩**한다.
  여기에 이미 인코딩된 Encoding 키를 넣으면 **이중 인코딩**되어 키가 깨지고 인증 오류가 난다.
  따라서 원본인 **Decoding 키**를 넣어야 코드가 한 번만 인코딩해 올바른 값이 된다.
- 💡 **헷갈리면 안전책**: 일단 `BUS_API_KEY`에 **Decoding 키**를 넣고 동작을 확인한다.
  만약 인증 오류가 계속되면 Encoding 키로 바꿔 시도한다. (둘 다 메모해두면 전환이 쉽다.)
- 참고: 최근 일부 API는 키를 **하나로 통합** 발급하기도 한다. 이 경우 그 하나를 그대로 쓰면 된다.

### A-8. 일일 트래픽 한도와 한도 초과 시 대응

- **개발계정**은 보통 **1,000 ~ 10,000 호출/일** 한도다. (TAGO 버스도착정보는 **10,000건/일**.)
- 공모전 테스트·시연 용도로는 충분하다.
- 한도를 더 늘리려면: 마이페이지 개발계정에서 해당 API의 **[운영계정 전환]** 또는 **[트래픽 변경 신청]** 을 진행한다.
  운영계정 전환에는 **활용사례(서비스 URL/설명) 등록**이 요구되며 별도 승인이 필요할 수 있다.

### A-9. 같은 계정/서비스키가 여러 API에 재사용되나?

- **계정은 공용**이다. data.go.kr에 한 번 가입하면 모든 API 신청에 같은 계정을 쓴다.
- **활용신청(서비스키 발급)은 API마다 따로** 해야 한다. 즉, 버스용·관광용을 **각각 활용신청**해야 각 키가 활성화된다.
- 발급되는 인증키 문자열 자체는 계정 단위로 동일한 값이 재사용되는 경우가 많지만,
  **해당 API에 활용신청이 등록돼 있지 않으면 같은 키로 호출해도 거부**된다.
  → 그래서 `BUS_API_KEY`와 `TOUR_API_KEY`에 **같은 문자열**을 넣게 되더라도, **두 API 모두 활용신청을 완료**해야 한다.

> 출처:
> - 공공데이터포털 메인/이용가이드: <https://www.data.go.kr/> , <https://www.data.go.kr/ugs/selectPublicDataUseGuideView.do>
> - Encoding/Decoding 키 차이(개발 블로그 정리): <https://velog.io/@naked_thunder/RestTemplate-API연동하기>
> - 활용신청 절차 정리: <https://www.elancer.co.kr/blog/detail/1045>

## B. `BUS_API_KEY` — 국토교통부 TAGO 버스도착정보 (1순위 권장)

전국 단위 실시간 버스 도착정보다. 서울 외 도시까지 한 API로 커버되므로 **1순위로 추천**한다.

### B-1. 활용신청

1. §A-2 로그인 상태에서 검색창에 **`TAGO 버스도착정보`** 입력 → **[오픈 API]** 탭.
2. **`국토교통부_(TAGO)_버스도착정보`** 를 클릭해 상세 페이지로 이동.
   - 직접 주소: <https://www.data.go.kr/data/15098530/openapi.do>
3. §A-4 절차대로 **[활용신청]** → 양식 작성 → 제출. (개발단계 자동승인이라 즉시 승인됨.)
4. §A-6 위치에서 **일반 인증키(Decoding)** 를 복사 → `.env`의 `BUS_API_KEY=` 뒤에 붙여넣는다.

이 API가 제공하는 오퍼레이션(상세 페이지에서 확인):
- **정류소별 도착예정정보 목록 조회**
- 정류소별 특정노선버스 도착예정정보 목록 조회
- **도시코드 목록 조회**

### B-2. ⚠️ 중요 — 버스 조회에 필요한 보조 정보(cityCode·nodeId·노선ID)

TAGO 버스도착정보를 호출하려면 **도시코드(`cityCode`)**, **정류소ID(`nodeId`)**, 때로 **노선ID**가 필요하다.
이 값들은 버스도착정보 API만으로는 다 못 얻으므로, **같은 포털에서 보조 API도 함께 활용신청**해두는 것을 권장한다:

1. 검색창에 **`TAGO 버스정류소정보`** → **`국토교통부_(TAGO)_버스정류소정보`** 활용신청
   - 주소: <https://www.data.go.kr/data/15098534/openapi.do>
   - 정류소 이름으로 **nodeId**(정류소 고유 ID)를 찾을 수 있다.
2. 검색창에 **`TAGO 버스노선정보`** → 해당 API 활용신청 (노선ID·경유 정류소 조회용)
3. **도시코드(cityCode)** 는 버스도착정보 API의 **"도시코드 목록 조회"** 오퍼레이션으로 얻는다(예: 서울 25, 부산 21 등).

> 즉, 버스 기능을 제대로 쓰려면 **버스도착정보 + 버스정류소정보(+ 버스노선정보)** 를 **각각 활용신청**한다.
> 모두 같은 서비스키로 호출하지만, API별 활용신청이 등록돼 있어야 한다(§A-9).

### B-3. (대안) 서울시 버스도착정보

서울만 대상으로 한다면 서울 열린데이터광장(TOPIS) 또는 data.go.kr의 서울시 버스도착정보를 쓸 수도 있다.
- data.go.kr 검색창에 **`서울특별시 버스도착정보`** 로 검색해 활용신청하는 방식은 §A와 동일.
- 서울 열린데이터광장(`data.seoul.go.kr`)에서 별도 가입·인증키를 받는 경로도 있다.
- 단, **전국 커버리지·단일 키 관리** 측면에서 **TAGO가 더 유리**하므로 이 프로젝트는 TAGO를 기본으로 한다.

> 출처:
> - TAGO 버스도착정보: <https://www.data.go.kr/data/15098530/openapi.do>
> - TAGO 버스정류소정보: <https://www.data.go.kr/data/15098534/openapi.do>
> - TAGO 버스위치정보(참고): <https://www.data.go.kr/data/15098533/openapi.do>
> - TAGO 안내(한국교통안전공단): <https://main.kotsa.or.kr/portal/contents.do?menuCode=01080800>

## C. `TOUR_API_KEY` — 한국관광공사 TourAPI 영문(EngService2)

외국인용 **영어 관광 데이터**(관광지·행사·숙박·맛집 등)를 제공한다.

### C-1. ⚠️ 반드시 "영문" 서비스를 골라야 한다

한국관광공사는 국문/영문/일문/중문 등 **언어별로 별도 API**를 운영한다.
외국인 사용자를 위한 영어 데이터를 받으려면 **영문 서비스(EngService2)** 를 신청해야 한다.
(국문 서비스를 신청하면 한국어 데이터만 나온다.)

### C-2. 활용신청

1. §A-2 로그인 상태에서 검색창에 **`한국관광공사 영문 관광정보`** 입력 → **[오픈 API]** 탭.
2. 결과 중 이름에 **`영문`** 과 **`EngService2`**(또는 영문 관광정보 서비스)가 들어간 항목을 클릭한다.
   - 국문/일문/중문 버전이 함께 뜨므로 **"영문"** 글자를 꼭 확인하고 고른다.
3. §A-4 절차대로 **[활용신청]** → 양식 작성 → 제출.
   - 활용목적 상세 예: `방한 외국인 대상 영어 관광정보 안내 서비스`
4. §A-6 위치에서 **일반 인증키(Decoding)** 를 복사 → `.env`의 `TOUR_API_KEY=` 뒤에 붙여넣는다.

> 💡 §A-9대로 data.go.kr 인증키 문자열은 버스용과 같은 값일 수 있으나,
> **영문 관광정보 API에 활용신청이 등록돼 있어야** 이 키로 관광 데이터를 받을 수 있다.

> 영문 서비스 관련 문의: 한국관광공사 데이터팀 `tourapi@knto.or.kr` / 포털 1566-0025

> 출처:

> - TourAPI 이용 안내: <https://www.2025tourapi.com/sub/sub01.html>

## D. `TRANSIT_API_KEY` — ODsay LAB 대중교통 길찾기

대중교통 환승·경로 검색(지하철+버스 복합 경로)을 제공한다. data.go.kr가 아닌 **별도 사이트**다.

### D-1. 회원가입

1. 주소창에 **`https://lab.odsay.com`** 입력 후 접속.
2. 오른쪽 위 **[회원가입]**(또는 [Sign up]) 클릭.
3. 이메일/비밀번호 입력, 약관 동의 후 가입. 이메일 인증 메일이 오면 링크 클릭으로 인증 완료.

### D-2. 로그인 후 애플리케이션(서비스) 등록 → API Key 발급

1. 로그인한다.
2. 상단 메뉴에서 **[Application] → [애플리케이션 등록]** 으로 이동.
3. 등록 양식 작성:
   - **애플리케이션 이름**: 최대 20자. 예) `KoreaTripConcierge`
   - **카테고리**: 여행/교통 등 적절한 항목 선택.
   - **서비스 유형**: **`Basic`(무료)** 선택. (개인·학생·5인 이하 스타트업 대상 무료)
   - **사용자 유형 / 서비스 플랫폼**: 안내에 따라 선택.
4. 플랫폼별로 **각 1개의 API Key**가 발급된다. 등록 가능한 플랫폼:
   - **URI**(최대 5개 등록 가능) · **서버(Server)** · **Android** · **iOS**
   - 이 프로젝트는 Node 서버에서 호출하므로 **서버(Server)용 키**(또는 URI 키)를 사용한다.
5. 발급 확인 위치: **[Application] → [내 애플리케이션] → [개요]** 에서 API Key 확인.
6. 발급된 키를 복사 → `.env`의 `TRANSIT_API_KEY=` 뒤에 붙여넣는다.

> ⚠️ **ODsay 키는 URL 인코딩에 민감**하다. 코드에서 한 번 인코딩하므로, data.go.kr와 마찬가지로 키 자체를 미리 인코딩하지 말고 발급된 원본 그대로 넣는다.

### D-3. 무료(Basic) 요금제 한도와 주의사항

- **호출 한도**: ODsay LAB 개편으로 무료 Basic은 **하루 약 200건**으로 변경되었다(과거 1,000건에서 축소).
  - 부족하면 **Basic Plus**(유료 옵션, 100건 단위 추가, 약 2만 원~) 또는 **Standard(유료)** 로 늘릴 수 있다.
- 무료 Basic은 기본 통계(최대 12시간 전 호출횟수 확인)와 **국문 응답** 중심이다. **다국어(영문 등) 응답은 유료(Standard)** 에서 제공된다.
- ⚠️ **약관상 상업적 사용 주의**: 무료 Basic은 개인/학생/소규모 스타트업 한정이다.
  공모전 출품·시연 단계는 가능하나, **상업적 서비스로 확장 시 유료 전환이 필요**할 수 있으니 운영정책을 반드시 확인한다.

> 출처:
> - ODsay LAB 이용 가이드: <https://lab.odsay.com/guide/guide>
> - ODsay LAB 운영정책(요금/한도): <https://lab.odsay.com/doc/totalPolicy>

## E. `.env` 설정 — 받은 키를 프로젝트에 넣기

1. 프로젝트 루트 폴더에 있는 **`.env.example`** 파일을 복사해 이름을 **`.env`** 로 바꾼다.
   - Windows 파일탐색기: `.env.example` 우클릭 → 복사 → 붙여넣기 → 이름을 `.env` 로 변경.
   - 또는 PowerShell에서: `Copy-Item .env.example .env`
2. `.env` 파일을 메모장 등 텍스트 편집기로 열고, 각 변수 뒤에 받은 키를 붙여넣는다(따옴표 없이, 공백 없이):

   ```dotenv
   BUS_API_KEY=여기에_TAGO_버스_Decoding_인증키
   TOUR_API_KEY=여기에_관광공사_영문_Decoding_인증키
   TRANSIT_API_KEY=여기에_ODsay_서버용_API_Key
   PORT=8080
   ```

   - 키 문자열에 줄바꿈이 들어가지 않도록 **한 줄로** 붙여넣는다.
   - `=` 좌우에 **공백을 넣지 않는다**.
3. 저장하고 서버를 재시작하면 키가 적용된다.

### E-1. ⚠️ 보안 — 반드시 지킬 것

- **`.env`는 절대 Git에 커밋하지 않는다.** (프로젝트 `.gitignore`에 이미 포함되어 있다 — 확인만 하고 건드리지 말 것.)
- 키를 **채팅 응답·로그·콘솔 출력·캡처 화면**에 노출하지 않는다. 공유 시에는 마스킹한다.
- 키를 외부 사이트(파스트빈, 깃허브 이슈 등)에 붙여넣지 않는다.
- 카카오 심사정책상 **개인정보 6종(주민/면허/여권/외국인등록/카드/계좌)은 수집·전송 금지**다.
  API 키는 이에 해당하지 않지만, 사용자 입력에서 위 정보를 받지 않도록 유지한다(docs/01 참조).
- 키가 노출되었다면 즉시 발급처(마이페이지 / ODsay Application)에서 **재발급(폐기 후 신규)** 한다.
