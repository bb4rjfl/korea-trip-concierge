# 12. 로컬 비즈니스 POI API 신청 가이드 (비개발자용 클릭 따라하기)

> **목적**: Korea Trip Concierge 가 방한 외국인에게 식당/카페/쇼핑 등 **외국인 친화 로컬 비즈니스**를
> 추천할 때, 한 곳의 데이터만으로는 커버리지가 부족하다. 그래서 **여러 POI(Point Of Interest) API**를
> 함께 신청해 데이터를 보강한다. 이 문서는 **개발 지식이 전혀 없어도** 각 사이트 화면에서
> 무엇을 클릭하면 되는지 단계별(1, 2, 3…)로 안내한다.
> 각 절 하단에 **출처 URL**을 달았다(2026년 6월 기준 실제 화면 확인). 사이트는 수시로 개편되므로
> 버튼 위치가 약간 다를 수 있다 — 그럴 땐 **굵게 표시한 키워드**로 찾으면 된다.
>
> 📎 이미 발급한 키 3종(`BUS_API_KEY` / `TOUR_API_KEY` / `TRANSIT_API_KEY`)은 **docs/08**을 참조.
> 이 문서(12)는 **POI 보강용 5종**을 다룬다.

## 0. 한눈에 보기 — 5개 API 요약표

| API | 발급처 | `.env` 변수 | 무료 한도 | 즉시 발급? |
|---|---|---|---|---|
| **① 네이버 지역검색** | 네이버 개발자센터 `developers.naver.com` | `NAVER_CLIENT_ID`, `NAVER_CLIENT_SECRET` | **일 25,000건** (지역검색은 쿼리당 **최대 5건** 반환) | ✅ 즉시(휴대폰 인증 후) |
| **② Foursquare Places** | Foursquare 개발자 콘솔 `foursquare.com/developers` | `FOURSQUARE_API_KEY` | 개발자 계정 **월 $200 크레딧** 제공 / 무료 Pro 호출 한도는 2026-06-01 개편 중(아래 §B-4 확인) | ✅ 즉시(이메일 가입 후) |
| **③ 서울 관광식당** | 서울 열린데이터광장 `data.seoul.go.kr` | `SUBWAY_API_KEY` 재사용 ❌ → **별도 `일반 인증키`** 필요(아래 §C-0) | 일반 인증키: 보통 **일 1,000건/키**, 한 번에 최대 1,000행 | ✅ 즉시(로그인 후 인증키 신청) |
| **④ 서울 모범음식점** | 서울 열린데이터광장 `data.seoul.go.kr` | ③과 **같은 `일반 인증키`** 사용 | ③과 동일 | ✅ 즉시 |
| **⑤ 면세점 / 외국인카드 매장** | (전용 데이터 없음) → 대안 §D | `DUTYFREE_API_KEY` *(또는 `TOUR_API_KEY` 재사용)* | 대안별 상이 | 대안별 상이 |

> ⚠️ **③·④ 핵심 주의**: 이 프로젝트가 이미 가진 `SUBWAY_API_KEY`는 서울 열린데이터광장의
> **"실시간 지하철 인증키"**(도메인 `swopenapi.seoul.go.kr`)다. 관광식당·모범음식점 같은 일반 데이터셋은
> **"일반 인증키"**(도메인 `openapi.seoul.go.kr`)를 쓴다. **두 키는 종류가 다르다.** 다만
> **같은 계정에서 둘 다 발급**되므로, 새로 가입할 필요 없이 일반 인증키만 추가로 신청하면 된다. (§C-0 상세)
>
> ⚠️ **⑤ 솔직한 결론**: "외국인 카드 결제 가능 매장"만 모은 공개 데이터는 **존재하지 않는다.**
> "사후면세점(택스리펀)" 전용 전국 오픈API도 사실상 없다. 따라서 **TourAPI 쇼핑/면세점 분류**를
> 재활용하는 것을 1순위 대안으로 권장한다. (§D 상세)

## A. ① 네이버 지역검색 API — `NAVER_CLIENT_ID` / `NAVER_CLIENT_SECRET`

네이버 검색 데이터 기반의 지역(업체·기관) 검색이다. 한국 식당/카페 커버리지가 넓고 **한글 상호·주소·전화·좌표**를
잘 준다. 외국인에게 보여줄 땐 우리 서버에서 영문 보강/로마자화를 거친다.

### A-1. 회원가입 / 로그인

1. 웹브라우저(크롬 권장) 주소창에 **`https://developers.naver.com`** 입력 후 접속한다.
2. 화면 오른쪽 위 **[로그인]** 클릭. 일반 네이버 아이디로 로그인하면 된다(없으면 먼저 네이버 회원가입).
3. 개발자센터를 처음 쓰면 **개발자 이용약관 동의** 화면이 한 번 나온다 — **[동의]** 후 진행.

### A-2. 애플리케이션 등록

1. 상단 메뉴 **[Application] → [애플리케이션 등록]** 을 클릭한다.
   (또는 첫 진입 시 **[Application 등록]** 안내 버튼.)
2. **이용약관에 동의합니다** 를 선택하고 **[확인]**.
3. (최초 1회) **계정 정보 등록** 단계가 나오면 **휴대폰 본인인증**을 완료하고, 요청하면 **회사 이름**(개인이면 임의 이름, 예: `Korea Trip Concierge`)을 입력한 뒤 **[확인]**.
4. **애플리케이션 이름**을 자유롭게 입력한다. 예) `KoreaTripConcierge`.
5. **사용 API** 항목에서 드롭다운/체크박스 중 **[검색]** 을 선택한다.
   - ⚠️ "지도", "로그인" 등이 아니라 반드시 **[검색]**(Search) 이어야 지역검색이 포함된다.
6. **비로그인 오픈 API 서비스 환경**(사용 환경) 에서 **[WEB 설정]** 을 선택한다.
   - **웹 서비스 URL**에 임의의 주소를 입력한다. 로컬 테스트면 `http://localhost` 또는 배포 URL을 넣어도 된다.
     (지역검색은 서버에서 호출하므로 이 값이 호출을 막지는 않지만, 칸이 비면 등록이 안 된다.)
7. 하단 **[등록하기]** 버튼을 클릭한다.

### A-3. Client ID / Client Secret 확인

1. 등록이 끝나면 자동으로 **[내 애플리케이션]** 화면으로 이동한다(또는 상단 [Application] → [내 애플리케이션]).
2. 방금 만든 앱을 클릭하면 **개요(Overview)** 에 **Client ID** 와 **Client Secret** 이 보인다.
   - **Client Secret**은 기본적으로 가려져 있을 수 있다 — 옆의 **[보기]**(눈 아이콘) 를 눌러 확인·복사.
3. 두 값을 복사해 `.env`에 넣는다(§E):
   - `NAVER_CLIENT_ID=` 뒤에 **Client ID**
   - `NAVER_CLIENT_SECRET=` 뒤에 **Client Secret**

### A-4. 무료 한도 · 호출 특성 (반드시 알아둘 것)

- **일 25,000건** 무료 호출 한도(검색 API 공통, 애플리케이션 단위). 공모전 시연·테스트엔 충분하다.
- **지역검색은 트래픽 보호를 위해 쿼리당 `display` 최대 5건**만 반환한다(많이 못 받음).
  → 한 번에 다섯 곳만 받으므로, 우리 서버는 **검색어/지역을 좁혀** 정확도를 높이는 방식으로 쓴다.
- 인증은 **HTTP 헤더**로 보낸다: `X-Naver-Client-Id`, `X-Naver-Client-Secret`.
  (서버 코드가 자동 처리하므로 사용자가 직접 신경 쓸 필요는 없다.)
- 요청 URL 형식(참고): `https://openapi.naver.com/v1/search/local.json?query=검색어&display=5&sort=random`

> 출처:
> - 네이버 개발자센터(애플리케이션 등록): <https://developers.naver.com/>
> - 등록 절차(검색 API → WEB 설정 → Client ID/Secret) 정리: <https://guide.bati.ai/service/api/naverapi>
> - 지역검색 display 최대 5건 / 헤더 인증 안내(개발 블로그): <https://velog.io/@cyseok123/Spring-네이버-지역-검색-API-사용하기>

## B. ② Foursquare Places API — `FOURSQUARE_API_KEY`

글로벌 POI 데이터. **영어 상호·카테고리**가 잘 정리돼 있어, 외국인에게 보여줄 때
네이버(한글) 대비 강점이 있다. 카테고리 체계가 풍부해 "외국인 친화 카페/레스토랑" 필터링에 유리하다.

### B-1. 개발자 계정 생성

1. 주소창에 **`https://foursquare.com/developers`** 입력 후 접속한다(또는 `location.foursquare.com` → Developers).
2. 오른쪽 위 **[Sign Up]**(또는 [Get Started] / [Log In]) 클릭.
3. 이메일/비밀번호로 가입하거나 소셜 로그인. 이메일 인증 메일이 오면 링크를 눌러 인증.

### B-2. 프로젝트(Project) 생성

1. 계정을 처음 만들면 **첫 프로젝트 생성**을 바로 요구한다. (이후에도 언제든 새 프로젝트 추가 가능.)
2. **Create a New Project** 화면에서 **프로젝트 이름**을 입력한다. 예) `KoreaTripConcierge`.
3. 안내에 따라 생성 완료. 콘솔(Developer Console) 좌측/상단에서 프로젝트를 선택할 수 있다.

### B-3. Service API Key 발급 (신형 방식)

> 신형 Foursquare는 예전의 `Client ID / Client Secret` 대신 **단일 Service API Key** 를 쓴다.
> 우리 `.env`의 `FOURSQUARE_API_KEY` 가 바로 이 Service Key다.

1. 콘솔에서 해당 **프로젝트**를 연 뒤 **[Settings]**(설정) 페이지로 이동한다.
2. **Service API Keys** 섹션에서 **[Generate Service API Key]** 버튼 클릭.
3. 팝업(modal)에서 **Service API Key Name**(식별용 이름, 예: `server`)을 입력하고 **[Next]**.
4. 생성된 키가 표시되면 **[Copy to Clipboard]**(복사) 아이콘으로 **즉시 복사**한다.
   - ⚠️ **이 키는 생성 직후 단 한 번만 전체가 표시된다.** 복사 후 안전한 곳에 보관하고 **[Dismiss]**.
   - 잃어버리면 다시 볼 수 없으니, 같은 섹션에서 **Remove(폐기) 후 재발급**해야 한다.
5. 복사한 키를 `.env`의 `FOURSQUARE_API_KEY=` 뒤에 붙여넣는다(§E).
   - 호출 시 인증은 보통 헤더 `Authorization: <SERVICE_KEY>` 로 보낸다(서버 코드가 처리).

### B-4. ⚠️ 무료 한도 — 2026년 전환기 주의

- Foursquare는 가격/무료 정책을 **2026-06-01부로 개편**했다. 시점·플랜에 따라 표기가 달라질 수 있어
  **반드시 콘솔의 Billing/Pricing 실제 표시값을 우선** 확인한다.
- 확인된 요점(2026년 6월 기준, 변동 가능):
  - **개발자 계정에 월 $200 사용 크레딧**이 제공된다(소규모 테스트·시연엔 충분).
  - **Pro 엔드포인트**(Place Search, Place Details, Autocomplete 등 기본 필드)에 무료/저단가 호출 구간이 있다.
  - **Premium 필드**(사진·팁·평점·영업시간 등)는 별도 과금이며 무료 구간이 없을 수 있다.
- 우리 서버는 기본 검색/상세(Pro) 위주로 쓰므로 **무료 크레딧 범위 내 운영**을 목표로 한다.
  Premium 필드는 꼭 필요할 때만 호출하도록 코드에서 관리한다.

> 출처:
> - 프로젝트 생성: <https://docs.foursquare.com/developer/docs/create-a-new-project>
> - Service API Key 발급/관리(Generate → 1회만 표시 → 복사): <https://docs.foursquare.com/developer/docs/manage-service-api-keys>
> - 개발자 콘솔 시작 가이드: <https://docs.foursquare.com/developer/docs/developer-console-get-started>
> - 요금/무료정책(2026-06-01 개편, $200 크레딧): <https://foursquare.com/pricing/> , <https://docs.foursquare.com/developer/reference/upcoming-changes>

## C. ③④ 서울 열린데이터광장 — 관광식당 · 모범음식점 (`일반 인증키`)

서울시가 운영하는 **열린데이터광장(`data.seoul.go.kr`)** 의 공개 데이터다.
**위생·서비스 우수 업소(모범음식점)**, **관광 편의 식당** 등을 데이터로 받아 외국인 추천에 보강한다.

### C-0. ⚠️ 먼저 — 우리 `SUBWAY_API_KEY` 를 그대로 못 쓰는 이유

서울 열린데이터광장의 인증키는 **두 종류**다(이 점이 data.go.kr과 다른 핵심).

| 키 종류 | 호출 도메인 | 용도 | 이 프로젝트 |
|---|---|---|---|
| **실시간 지하철 인증키** | `swopenapi.seoul.go.kr` | 실시간 지하철 도착/위치 | ✅ 이미 발급 = `SUBWAY_API_KEY` |
| **일반 인증키** | `openapi.seoul.go.kr` | 그 외 모든 일반 데이터셋(식당 등) | ❌ **아직 없음 → 새로 신청 필요** |

- 즉, `SUBWAY_API_KEY`(지하철용)로는 **관광식당·모범음식점 데이터를 호출할 수 없다.**
- 단, **같은 계정**에서 두 키를 모두 발급한다. 이미 가입돼 있으므로 **새 가입은 불필요**하고,
  **"일반 인증키"만 추가로 신청**하면 된다.
- ✅ **한 번 발급한 일반 인증키 하나로 열린데이터광장의 (거의) 모든 일반 오픈API를 호출**할 수 있다.
  data.go.kr처럼 **데이터셋별 "활용신청"을 따로 할 필요가 없다.** → 관광식당·모범음식점 둘 다 같은 키로 OK.

### C-1. 일반 인증키 신청 (1회)

1. **`https://data.seoul.go.kr`** 접속 → 오른쪽 위 **[로그인]**.
   (열린데이터광장 계정이 없으면 안내에 따라 서울시 통합 회원가입 후 로그인. 지하철 키를 받았다면 같은 계정으로 로그인.)
2. 상단/마이페이지에서 **[인증키 신청]** 으로 이동.
   - 직접 주소: <https://data.seoul.go.kr/together/mypage/actkeyMain.do>
3. 키 종류 중 **[일반 인증키 신청]** 을 선택한다(‎"실시간 지하철 인증키"가 아니라).
4. 신청 양식 작성:
   - **활용 용도/서비스명**: 예) `방한 외국인 대상 여행 안내 MCP 서비스`
   - **사용 URL/설명** 등 요청 항목 입력.
5. **[신청]** 제출 → 보통 **즉시 발급**된다. 발급된 긴 문자열이 곧 인증키다.
6. 발급 확인: 마이페이지 **[인증키 신청/관리]** 에서 언제든 다시 볼 수 있다.
7. `.env`에 넣는다(§E):
   - 새 변수로 관리하려면 `SEOUL_API_KEY=`(별칭)에 넣고, 코드가 참조하는 키 이름과 맞춘다.
   - 또는 명확성을 위해 **지하철 키(`SUBWAY_API_KEY`)와 별도 줄**로 일반 인증키를 보관한다.
   - ⚠️ **지하철 키 칸에 일반 인증키를 덮어쓰지 말 것** — 둘은 용도가 다르다.

### C-2. ③ 관광식당 데이터셋 찾기

1. `data.seoul.go.kr` 상단 **검색창**에 **`관광식당`** 입력 후 검색.
2. 결과에서 데이터셋 제목을 클릭해 **상세 페이지(datasetView)** 로 들어간다.
3. 상세 페이지 상단의 **제공 형태** 탭을 확인한다: **`시트` / `차트` / `Open API`** 중 **[Open API]** 탭이 있어야
   실시간 호출이 가능하다. (Open API 탭이 없고 시트/파일만 있으면, 그 데이터셋은 다운로드용이다 → §C-4 참고.)
4. **[Open API]** 탭에서 **서비스명(SERVICE)**, **샘플 URL**, **요청/응답 변수(출력값)** 를 확인한다.
   - 호출 형식(공통): `http://openapi.seoul.go.kr:8088/{인증키}/json/{SERVICE}/{시작}/{끝}/`
   - 여기 `{SERVICE}` 자리에 상세 페이지에 적힌 **서비스명**을 넣는다(데이터셋마다 다름).
5. ⚠️ **솔직한 안내**: "관광식당"이라는 **정확한 단일 명칭의 서울 오픈API가 항상 있는 것은 아니다.**
   검색 결과가 비거나 시트 전용이면, **모범음식점(§C-3)** 또는 TourAPI 음식 분류로 대체한다.
   (검색 시 `관광 음식점`, `음식문화`, `안심식당` 등 유사 키워드도 함께 시도해 본다.)

### C-3. ④ 모범음식점 데이터셋 찾기 (⚠️ 자치구별로 나뉨)

1. 검색창에 **`모범음식점`** 입력 후 검색.
2. ⚠️ **중요**: 서울 모범음식점은 **하나의 통합 시(市) 단위 API가 아니라 자치구(구)별 데이터셋**으로 제공된다.
   예시(실제 존재 확인됨):
   - 서울시 **동작구** 모범음식점 지정 현황 — `OA-10602` (<https://data.seoul.go.kr/dataList/OA-10602/S/1/datasetView.do>)
   - 서울시 **강서구** 모범음식점 지정 현황 — `OA-9986`
   - 서울시 **마포구** 모범음식점 지정 현황 — `OA-11372`
   - (강서구 취소현황 `OA-10004`, 은평구 취소현황 `OA-10543` 등 "취소" 데이터셋과 혼동 주의)
3. 각 구 데이터셋 상세에서 **[Open API]** 탭 유무와 **서비스명(SERVICE)** 을 §C-2-4처럼 확인한다.
   - 일부 구는 **시트/차트 전용**(Open API 미제공)일 수 있다 → 그 구는 파일 다운로드로 대체하거나 제외.
4. 우리 서버는 **외국인 방문 빈도가 높은 구**(예: 중구·종로·마포·강남 등)부터 우선 연동하는 것을 권장한다.
   모든 구를 다 넣을 필요는 없다.

### C-4. 데이터셋이 "시트/파일"만 제공할 때 (Open API 없음)

- 일부 데이터셋은 **다운로드(CSV/JSON)** 만 제공하고 실시간 Open API가 없다.
- 이 경우 두 가지 선택:
  1. **정적 스냅샷**으로 받아 서버에 내장(주기적 수동 갱신). 변동이 적은 "지정 현황"엔 무리 없음.
  2. 같은 주제의 **Open API 제공 데이터셋**(다른 구/다른 기관)으로 대체.
- 어느 쪽이든 **개인정보 6종은 수집·전송 금지** 원칙(상호·주소·좌표 등 공개 영업정보만 사용).

> 출처:
> - 인증키 신청 페이지: <https://data.seoul.go.kr/together/mypage/actkeyMain.do>
> - Open API 이용 안내(일반 vs 실시간 지하철 인증키, 도메인 차이, 1회 최대 1,000행): <https://data.seoul.go.kr/together/guide/useGuide.do>
> - 모범음식점(구별) 예시 데이터셋: <https://data.seoul.go.kr/dataList/OA-10602/S/1/datasetView.do> , <https://data.seoul.go.kr/dataList/OA-9986/S/1/datasetView.do> , <https://data.seoul.go.kr/dataList/OA-11372/S/1/datasetView.do>
> - 공공데이터 목록: <https://data.seoul.go.kr/dataList/datasetList.do>

## D. ⑤ 면세점 / 외국인 카드 가능 매장 — `DUTYFREE_API_KEY` (또는 `TOUR_API_KEY` 재사용)

### D-0. ⚠️ 솔직한 현실 진단 (추정 아님, 실제 검색 결과)

- **"외국인 카드 결제 가능 매장"만 정리한 공개 데이터(오픈API)는 존재하지 않는다.**
  (해외카드 가맹 여부는 가맹점·카드사 영역이라 공공데이터로 개방돼 있지 않다.)
- **"사후면세점(택스리펀, Tax Refund)" 전국 단위 오픈API**도 사실상 없다.
  data.go.kr에서 "사후면세점/택스리펀/외국인 면세판매장"으로 검색해도 **호출용 오픈API는 확인되지 않았다**
  (수출입 통계 등 관세청 무역 API만 나옴).
- 검색으로 **실재가 확인된** 면세 관련 데이터는 다음 정도다(대부분 통계/파일, 지역 한정):
  - **JDC(제주) 지정면세점 매장정보** — 제주국제자유도시개발센터, **Open API** 제공(제주 한정).
  - 인천국제공항공사 "여객기 운항편 조회(면세점용)" — 공항 운항정보(매장 목록 아님).
  - 제주 "내국인 관광객 업종별 카드 이용데이터"(면세점 포함) — 통계성.

### D-1. ✅ 1순위 권장 — TourAPI "쇼핑(면세점)" 분류 재활용 (`TOUR_API_KEY`)

가장 현실적이고 외국인 친화적인 방법. **이미 발급한 `TOUR_API_KEY`(한국관광공사 영문)로 추가 비용·신청 없이** 면세점을 다룬다.

1. TourAPI(영문 EngService2)의 **지역기반 관광정보 / 위치기반 관광정보** 오퍼레이션을 사용한다.
2. **콘텐츠 타입(contentTypeId)** 중 **쇼핑(Shopping)** 분류를 지정해 조회하면 **면세점(Duty Free)** 매장이 포함된다.
   - 분류코드(cat1/cat2/cat3) 체계상 쇼핑 하위에 면세점 항목이 있다. 정확한 코드값은 TourAPI의
     **서비스분류코드/분류체계코드** 오퍼레이션으로 조회해 확정한다(언어=영문).
3. 영문 데이터이므로 외국인에게 **영어 상호·주소·설명**을 바로 제공할 수 있다.
4. `.env` 처리: **별도 키 불필요.** `DUTYFREE_API_KEY`를 비워두고 코드가 `TOUR_API_KEY`를 재사용하도록 한다.
   (변수 분리를 원하면 `DUTYFREE_API_KEY`에 같은 TourAPI 키 문자열을 넣어도 동작은 동일.)

### D-2. (선택) 제주 한정 면세 데이터 — data.go.kr

제주 여행 시나리오를 강화하려면 JDC 데이터를 추가할 수 있다(전국 범위 아님).

1. **`https://www.data.go.kr`** 로그인(이미 계정 있음 — docs/08 참조).
2. 검색창에 **`JDC면세점`** 또는 **`지정면세점 매장정보`** 입력 → **[오픈 API]** 탭.
3. **`제주국제자유도시개발센터_제주국제공항 JDC면세점 매장정보`** 상세 → **[활용신청]**
   (data.go.kr는 §A 방식대로 **데이터셋별 활용신청** 필요 — docs/08 §A-4 참고).
4. 발급 인증키를 `DUTYFREE_API_KEY`에 넣는다(또는 기존 data.go.kr 키 재사용 + 이 API 활용신청만 추가).

### D-3. (대안) 사후면세/택스리펀 안내는 "지식 응답"으로 대체

전용 매장 데이터가 없으므로, **택스리펀 절차·환급 한도·공항 환급 위치** 같은 **설명형 안내**는
큐레이션 지식 툴(현재 `explainPayment`의 백화점·면세 분기, docs/00 참조)로 제공하는 것이 솔직하고 안정적이다.
"이 매장이 외국인 카드가 되는가?"를 단정하지 말고, **카드 브랜드(Visa/Master/UnionPay) 수용 여부는
현장 확인**을 안내하도록 응답을 설계한다(과장·오정보 방지 — docs/01 심사정책).

> 출처:
> - data.go.kr "면세점" 검색(JDC 매장정보 Open API / 통계 파일 위주, 사후면세 전용 API 없음): <https://www.data.go.kr/tcs/dss/selectDataSetList.do?keyword=면세점>
> - 한국관광공사 관광정보 서비스(쇼핑 분류 포함, 15종 콘텐츠): <https://www.data.go.kr/data/15101578/openapi.do> , <https://api.visitkorea.or.kr/>
> - 관세청 오픈API 목록(무역통계 중심, 사후면세판매장 호출 API 미확인): <https://www.data.go.kr/>

## E. `.env` 에 키 넣기

1. 프로젝트 루트의 **`.env`** 파일을 텍스트 편집기로 연다(없으면 `.env.example` 복사 → `.env`로 이름 변경; docs/08 §E).
2. 해당 변수 뒤에 키를 **한 줄로, 따옴표·공백 없이** 붙여넣는다:

   ```dotenv
   # ① 네이버 지역검색
   NAVER_CLIENT_ID=여기에_네이버_Client_ID
   NAVER_CLIENT_SECRET=여기에_네이버_Client_Secret
   # ② Foursquare Places
   FOURSQUARE_API_KEY=여기에_Foursquare_Service_API_Key
   # ③④ 서울 열린데이터광장 — 일반 인증키 (지하철 키와 별개!)
   SUBWAY_API_KEY=여기에_서울_실시간지하철_인증키   # (기존, 지하철 전용 — 건드리지 말 것)
   SEOUL_API_KEY=여기에_서울_일반_인증키            # (신규, 관광식당/모범음식점용)
   # ⑤ 면세점 — 기본은 TOUR_API_KEY 재사용. 별도 쓸 때만 입력.
   DUTYFREE_API_KEY=
   ```

   - ⚠️ `=` 좌우에 **공백 금지**, 키 안에 **줄바꿈 금지**.
   - ⚠️ **`SEOUL_API_KEY`(일반)와 `SUBWAY_API_KEY`(지하철)는 다른 값**이다. 헷갈려 덮어쓰지 말 것.
   - `.env.example`에는 **빈 변수만** 두고 실제 값은 절대 적지 않는다(아래 보안 규칙).
3. 저장 후 서버를 재시작하면 적용된다.

> 📌 코드가 참조하는 변수명이 `SEOUL_API_KEY`인지 `SUBWAY_API_KEY`인지는 구현에 따른다.
> **이 문서는 코드를 바꾸지 않는다.** 실제 연동 시, 코드가 읽는 변수명에 맞춰 키를 넣거나
> 변수명을 코드와 일치시키는 작업을 별도 PR에서 진행한다(R-DOC: 그때 본 문서·`07_progress`도 함께 갱신).

### E-1. ⚠️ 보안 — 반드시 지킬 것 (docs/08 §E-1 재확인)

- **`.env`는 절대 Git에 커밋하지 않는다**(`.gitignore`에 이미 포함 — 확인만).
- 키를 **채팅·로그·콘솔·캡처**에 노출하지 않는다. 공유 시 마스킹.
- **개인정보 6종(주민/면허/여권/외국인등록/카드/계좌)은 수집·전송 금지**(카카오 심사정책, docs/01).
  특히 ⑤ 면세/카드 기능에서 **사용자 카드번호를 절대 입력받지 않는다.**
- 키가 노출되면 즉시 발급처에서 **폐기 후 재발급**(네이버 앱 삭제·재생성 / Foursquare Remove→Generate / 서울 인증키 재신청).

## F. 발급 후 점검 체크리스트

- [ ] **① 네이버**: 개발자센터 로그인 → 애플리케이션 등록(**검색** API, **WEB**) → Client ID/Secret 복사 → `.env`
- [ ] **② Foursquare**: 개발자 계정 → 프로젝트 생성 → Settings에서 **Service API Key 발급(1회만 표시!)** → `.env`
- [ ] **③④ 서울**: 로그인 → **[일반 인증키] 신청**(지하철 키 아님) → `관광식당`/`모범음식점` 검색해 **Open API 탭·서비스명 확인** → `.env`(지하철 키와 별도 줄)
- [ ] **⑤ 면세점**: 기본은 **`TOUR_API_KEY` 재사용(TourAPI 쇼핑/면세점)** — 별도 키 불필요. (제주 강화 시에만 JDC 활용신청)
- [ ] `.env`가 Git 추적 대상이 아님을 확인(`.gitignore`)
- [ ] 서버 재시작 후 각 POI 소스가 "미연동"이 아닌 실데이터를 반환하는지 확인
- [ ] (서울) 일부 데이터셋이 시트/파일 전용이면 §C-4 대체전략 적용 여부 결정
