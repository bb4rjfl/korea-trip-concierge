- **종료**: `/handoff` → `07_progress.md`·`06_decision_log.md` 갱신, 다음 할 일 명시.

## 2. 문서 최신화 규칙 (절대)
- **R-DOC-1**: 규칙/스펙/툴/데이터소스/결정이 바뀌면 **같은 변경에서** 해당 docs를 갱신. 문서 갱신 없는 코드 변경 금지.
- **R-DOC-2**: 결정은 `06_decision_log.md`에 `D-00x`로 기록(무엇을·왜·대안).
- **R-DOC-3**: 진행상황은 `07_progress.md`가 **단일 진실 소스(SSOT)**. 체크박스로 관리.
- **R-DOC-4**: `CLAUDE.md`의 "현재 상태"와 `07`이 어긋나면 `07` 기준으로 `CLAUDE.md` 수정.

## 3. 코딩 규칙
- **언어**: TypeScript(권장) + MCP 공식 SDK, Streamable HTTP, **stateless**.
- **툴 등록**: 각 툴은 `03_tool_contracts.md`의 inputSchema·annotations(5종)·영문 description(≤1024, 서비스명 포함) 그대로.
- **응답 빌더**: 모든 툴은 공통 `renderMarkdown()` + `buildChoiceFooter()`(04) 사용. **24k 가드 필수**.
- **외부 API**: 타임아웃(예 2.5s)·캐싱·재시도 1회. 실패 시 친화적 Markdown 에러+재시도 칩.
- **네이밍 린트**: 서버명·툴명에 `kakao`(대소문자 불문) 들어가면 **빌드 실패**하도록 사전 스크립트.
- **시크릿**: API 키·토큰은 `.env`(환경변수). 저장소·응답·로그에 노출 금지. 개인정보 6종 미처리.
- **성능**: 평균 100ms / p99 3,000ms 목표. 무거운 호출은 캐시 우선.
