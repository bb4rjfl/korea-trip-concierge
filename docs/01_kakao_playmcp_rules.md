## 3. ⭐ 서버 개발가이드 (반려 방지 필수)

### 3-1. 서버 생성 조건
- MCP 버전 **최소 `2025-03-26` ~ 최대 `2025-11-25`**
- **Streamable HTTP** 만 지원
- **Remote MCP** 만 지원 (공개 URL 접근 가능 도메인)
- **Stateless 권장** (no session)
- 인증 필요 시 **OAuth 또는 커스텀 헤더**
- **MCP Inspector** 로 사전 점검
- 활발히 운영되는 **공식 SDK** 사용/참조
- ❌ 서버명·툴명에 **`kakao`** 금지 (대소문자·prefix/suffix/중간 모두)

### 3-2. Tool 구성
- **툴 이름**: 1~128자, `A-Z a-z 0-9 _ -` 만, 중복 금지, **대소문자 구분**
- **툴 개수**: 최대 20 (초과 금지), **3~10개 권장**
- **필수 property**: `name`, `description`, `inputSchema`, `annotations`
- **annotations 5종 모두 값 지정**: `title`, `readOnlyHint`, `destructiveHint`, `openWorldHint`, `idempotentHint`
- **description**: **영문 권장**, **1,024자 이내**, **서비스명 포함**(고유명사 영문+국문 병기)
  - 예: `... from Korea Trip Concierge(코리아 트립 컨시어지)`

### 3-3. 응답 형식
- PlayMCP AI채팅: **`TextContent` 타입만** 허용 (도구함으로 Claude/ChatGPT 직접 연결 시 타입 확장 가능)
- **Markdown 권장**, API JSON 원문 금지(정제 필수)
- 이미지: `![설명](https://.../img.png)` 형태로 URL 삽입
- **Response 24k 초과 → 에러/반려.** result 최소화
- ※ PlayMCP는 현재 MCP 스펙의 **Resource·Prompt 미지원** → **Tool 중심 설계**

### 3-4. OAuth (개인정보 연동 시)
- Redirect URI: `https://playmcp.kakao.com/api/v1/applied-mcps/{mcpId}/authorize/oauth:callback`
- 인증정보 없음/만료 → **401** 응답
- 개인정보 카카오 전달 시 '개인정보 제3자 제공 동의' 화면 권장
- ※ Korea Trip Concierge는 **인증 불필요 설계 지향**(외국인 진입장벽 최소화). OAuth는 가급적 회피.

### 3-5. 운영(성능)
- **평균 100ms / p99 3,000ms 필수**
- ❌ 광고 노출 유도 금지

## 4. MCP 심사 정책 (반려 사유)

- Tool ≥1개 (권장 3~20)
- 표준 스펙 위배·비정상 동작 → 반려
- 제3자 서비스 인증정보 사용 → 관계 따라 반려
- ❌ Zapier 등 자동생성 MCP 등록 불가
- ❌ 명칭·출력만 바꾼 동일구조 MCP 반복 불가
- ❌ **"LLM 웹검색으로 충분히 구현 가능한 기능"만 → 반려/공개제한** (확장·추가 목적 명확하면 예외)
- 추상적·모호한 이름/설명 → 수정 요청
- ❌ 도구 실행에 유료 결제/구독 필수 → 반려
- 느림·timeout·과도한 리디렉션/크롤링/외부호출 → 반려
- 아웃링크 OK, **상업 링크·구매유도·리워드 과다 → 반려**
- 비속어/정치/성적/사회통념 위배 → 반려
- 악성 파일 다운로드 → 반려/비공개
- **개인정보 6종 수집·전송 → 반려**: 주민등록번호/운전면허번호/여권번호/외국인등록번호/카드번호/계좌번호
- 대표 이미지: 움직이는 이미지 불가, 서비스 성격 부합·고품질
- 인증 없음/만료 → 401 필요

## 5. 배포 (PlayMCP in KC, `https://playmcp.kakaocloud.io`)

- 계정당 **최대 2대**. PlayMCP 가입 카카오계정 로그인 필요.
- **방법 A — Git 소스 빌드(기본)**: 서버명/설명/**Git URL(루트에 Dockerfile 필수)**/브랜치(`main`)/Dockerfile 경로/PAT(private만) → 등록 → `Active` → Endpoint URL 복사
- **방법 B — 컨테이너 이미지**: **`linux/amd64` 필수**(arm64 실패) / Registry 호스트(`ghcr.io` 등)·사용자·비번(private만)·image_name·image_tag
- 삭제 복구 불가.

## 6. 등록·심사·공개 절차 (PlayMCP)
임시 등록 → "정보 불러오기" 성공 확인 → 도구함 추가 → AI채팅 테스트 → **대화 예시 3개 입력** → 심사 요청 → 승인 → **"전체 공개" 전환** → 상세 URL 복사 → 공모전 비즈폼 "Player 예선 참여"(최대 2개, **1회 제출**).

## 7. "버튼/선택지로 이어가기" 구현 근거 (UX 핵심과 연결)

- MCP에는 **elicitation**(서버가 선택지 스키마를 띄워 사용자가 고르게 함)이 있다. form mode = `2025-06-18`, URL mode = `2025-11-25`. **단, 클라이언트가 지원해야** 동작하며 PlayMCP AI채팅 지원은 보장되지 않는다.
- **따라서 예선 기본 전략**: 응답 본문(Markdown TextContent)에 **다음 액션 선택지를 명시**해 LLM이 자연스럽게 후속 툴콜하도록 유도(= 사용자는 짧게 누르듯 답함). 자세한 패턴은 `04_ux_interaction.md`.
- **본선(Kakao Tools)**: Widget/elicitation 등 향상된 인터랙션을 추가 검토. (7/30~8/27)
