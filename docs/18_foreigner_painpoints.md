## B. 4개 에이전트가 **독립적으로 일치**시킨 3대 킬러 차별점
- **A. "여기서 내 해외카드 될까?"** — 상황별 예측(키오스크 실패/대형체인 OK/온라인 ID차단/캐시리스버스 T-money) + 폴백. 누구도 영어로 상황형 답을 안 줌.
- **B. 한국 본인인증 장벽 내비게이터** — 예약·배달·택시·쇼핑·가입이 거의 다 PASS/카톡 본인인증으로 막힘 → "이건 한국ID 필요, 외국인용 대안은 X"를 **생태계 전체로** 매핑. 단일 리소스 부재.
- **C. 공휴일 인식 "지금 열렸나"** — 구글 open-now가 추석/설·한국공휴일을 놓침. 영어인터넷이 stale.

## C. 구현 로드맵 (컨셉·규칙 안전)

### 🆕 신툴 후보: `explainKoreanService` (본인인증/시스템 장벽 내비게이터)
- input: service enum(taxiApp / foodDelivery / reservation / onlineShopping / kakaoNaverSignup / simEsim / taxRefund / entryDocs / emergencyMedical / kiosk)
- output: **막히는 지점 + 외국인용 정확한 우회**(Kakao T→"Pay to Driver"·k.ride/Uber / 배달→Shuttle·Baemin무가입 / 예약→CatchTable Global / 쇼핑→글로벌몰 / 카톡→이메일가입 / 세금환급 결정트리 / e-Arrival 절차 / 119·1339·1330).
- **순수 큐레이션**(D-009 안전·PII無·요청응답·LLM이 신뢰성있게 못줌). **컨셉 정체성 그 자체.** → 툴 11→12 (D-006 검토 필요).

### 🔧 기존 툴 강화 (신툴 없이)
1. **explainPayment** ← #1/#4/#10: 카드 상황매트릭스 + DCC + T-money(태그오프/환승/환불) + K-Pass/기후동행(체류일별) + 부가세환급 결정트리 + 택시 "Pay to Driver"/공정요금/1330.
2. **getNowInfo/landmarks** ← #5: **한국 공휴일 캘린더(큐레이션)** → "추석 휴무" 판정.
3. **translateMenuContext** ← #7/#13: 할랄/채식 "안전" 플래그 + 숨은 멸치·생선육수·김치젓갈 경고 + 한국어 식이/알레르기 카드 텍스트 + 주문문장 확대.
4. **findForeignerFriendlyStore** ← #12/#9: 응급(119/1339/1330)·심야약국 + **1330 상시 CTA** + ATM DCC/4자리PIN.
5. **getTransitRoute** ← #2/#6: 목적지 **한국어명 "네이버에 붙여넣기" 블록** + (큐레이션) 주요역 출구번호 + 택시 안티스캠.

### 🚫 비차별/규칙주의
- 에티켓·일반정보(#14)는 LLM이 잘함 → 가이드 푸터로만 가볍게.
- 못하는 것(정직 고지): 실제 예약/결제/티켓팅, 단말기에 카드 통과, 라이브 보행내비, 사진 메뉴 OCR, 주방 알레르겐 보증. → 우리 역할 = **사전무장 + 구조화 영어설명 + 라이브 open/transit/air**.
