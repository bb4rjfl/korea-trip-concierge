# 04. UX 인터랙션 — "버튼/선택지로 이어가기"

> Kev 요구: 사용자가 매번 긴 텍스트를 다시 안 쳐도, **버튼(새로고침처럼)이나 선택지를 미리 줘서 누르기만** 하면 이어지도록.
> 제약: PlayMCP AI채팅은 `TextContent`만 보장. **그래서 "선택지를 응답에 명시"하는 방식이 기본 전략.**

---

## 1. 두 가지 구현 경로

### (A) 기본 전략 — "응답 내 팔로업 질문 칩" (예선·전 클라이언트 호환) ✅
모든 툴 응답 **마지막에** 다음 액션을 **자연어 팔로업 질문**으로 노출한다. 사용자는 그 문구를 그대로 말/탭하면 되고, 호스트 LLM이 그 의도를 받아 **다음 툴을 자동 호출**한다. → 체감상 "버튼 누르기".

표준 푸터 형식(Markdown, 2026-06-29 개정):
```
---
**💬 You can ask me next / 다음으로 물어보세요:**
- 🚇 **How do I get to Seongsu?** / Seongsu 가는 길
- 🔎 **What are Seongsu's famous spots & shops?**
- 🗺️ **What's a popular Seongsu course for my style?**
```
- 칩은 **2~4개**, 각 칩 = (이모지 + **볼드 질문**[+ 한국어 병기]).
- 질문은 **다음 툴콜로 바로 매핑**되고 **맥락(동네·페르소나·장소명)을 담아 구체적**으로.
- 영문 우선 + 한국어 병기.

> ⚠️ **렌더링 현실(2026-06-29 실측, 카카오 AI채팅 스크린샷)**: 카카오 호스트는 우리 TextContent를 **그대로 보여주지 않고 LLM이 재작성(paraphrase)**해서 사용자에게 낸다(큐레이션 blurb가 다른 문장으로 바뀌어 나옴). 이때 **"Tap to continue" 같은 UI-메타데이터성 푸터는 통째로 누락**될 수 있다. → 그래서 칩을 ① **백틱/메타 제거**하고 ② **자연어 팔로업 질문**으로(호스트가 "유용한 다음 질문"으로 인식해 surface) ③ **맥락 구체화**로 바꿨다. 코드 칩은 항상 유지하되 wording이 호스트 재작성에서 살아남게 설계. (이미지 마크다운은 호스트가 깨진 텍스트로 보여 N11에서 제거; 공유는 클라이언트 기능[캡처/공유 툴 기존 제공].)

### (B) 강화 전략 — MCP elicitation (본선 Kakao Tools 검토용) ⏳
- 서버가 `elicitation/create`로 **선택지 스키마(enum)** 를 띄워 사용자가 클릭 → 진짜 버튼 UX.
- form mode = MCP `2025-06-18`, URL mode = `2025-11-25`. **클라이언트 지원 필요** → PlayMCP AI채팅 보장 안 됨.
- **예선엔 의존하지 않는다.** 본선(7/30~8/27) Widget과 함께 지원 여부 확인 후 도입.

> 결정: 예선은 (A) 100%. (B)는 본선 옵션. → `06_decision_log.md` D-003.

## 2. "조회로 알림 대체" 패턴 (K-Bus 핵심)
푸시 불가이므로, 사용자가 **한 번 묻고 → 칩으로 계속 새로고침**하게 만든다.

예시 흐름:
1. 사용자: "143번 탔고 명동성당에서 내릴래"
2. `trackBusArrival` →
   ```
   🚌 **Bus 143 → 명동성당(Myeongdong Cathedral)**
   현재 **3 stops away**, 약 **6 min**. 명동성당은 종로2가 다음다음입니다.

   ---
   **누르듯 골라주세요 / Tap to continue**
   - 🔄 `다시 확인` / `Refresh` — 위치 업데이트
   - 🚏 `거의 다 왔어?` / `Am I close?` — 1정거장 전 재확인
   - 🗺️ `내린 다음 길찾기` / `Route after I get off`
   ```
3. 사용자가 `다시 확인` 누르듯 입력 → 같은 툴 재호출. (세션 없이 stateless로 매번 최신)

---

## 3. 툴 간 연결(칩이 다른 툴을 부른다)
- `getAreaGuide` 끝 → `🍜 근처 맛집` → `findForeignerFriendlyStore`
- `findForeignerFriendlyStore` 끝 → `🚌 거기 가는 길` → `getTransitRoute`
- `translateMenuContext` 끝 → `🗣️ 주문 문장 만들기`
- → 자연스러운 **여정 흐름**을 칩으로 연결해 본선 투표에서 "끊김 없는 데모" 인상.

## 4. 작성 규칙
- 칩 푸터는 **항상 응답 24k 예산 안에서**. 본문이 길면 본문을 줄이고 칩은 유지.
- 칩 명령어는 **모호하지 않게**(심사: 모호한 표현 수정 요청 대상).
- 광고·구매유도·리워드 칩 금지(반려).
- 영문 우선(타깃 외국인), 한국어 병기.

## 5. 구현 헬퍼 (권장)
`buildChoiceFooter(choices: {emoji, cmd, cmdEn, desc, descEn}[])` 유틸을 만들어 모든 툴이 동일 포맷으로 칩을 생성. 테스트로 칩 2~4개·24k 가드 검증.
